Introduction

Internet

Système d'interconnexion de machines qui constitue un réseau informatique mondial, utilisant un ensemble standardisé de protocoles de transfert de données.

World Wide Web (WWW, le Web)

Système de documents liés entre eux et accessibles à travers internet.

Programmation Web

Développement de documents pour le web.

Introduction

Documents web

  • Page web : un document.
  • Site web : une collection de pages web.
  • Application web : page / site web + interactivité + logique applicative.

Outils du web

Languages (HTML, CSS, JavaScript), Protocole HTTP, Serveur, Navigateur (client).

Introduction

Pages statiques

  • Contenu sur disque = contenu chargé par le navigateur.
  • Aucune modification au contenu côté serveur.
  • N.B. Le contenu peut être modifié côté client (JavaScript).

Pages dynamiques

  • Contenu sur disque ≠ contenu chargé par le navigateur.
  • Le contenu est généré dynamiquement (généralement à partir d'une BD et en fonction de paramètres fournis avec la demande).

Introduction

Programme client

  • Code exécuté sur le navigateur.
  • Peut modifier le contenu en temps réel.
  • Ne nécessite pas une connexion avec le serveur.
  • Langage: JavaScript.

Programme Serveur

  • Code exécuté sur le serveur.
  • Ne peut modifier le contenu que lors du chargement d'une page à partir du serveur.
  • Langages: PHP, Ruby, Java, .Net (et tout autre langage).

Outils de Développement

Editeurs

  • Editeurs de texte simples (pas traitement de texte!).
  • Editeurs de code source (coloration syntaxique, mise en forme ...).
  • EDIs (erreurs de syntaxe, insertion automatique de code, exécution de code, débogage ...).
  • Editeurs WYSIWYG (différences de rendu avec les navigateurs, code généralement pas optimisé)
  • Exemples: Notepad++, Sublime Text 2, gEdit, Eclipse WTP, PHPStorm (commercial).

Outils de Développement

Serveurs Web

Apache, Nginx, IIS

Serveurs de BD

PostgreSQL, MySQL, MariaDB, Oracle, MongoDB ...

Environnements Complets

LAMP, WAMP, MAMP, LAPP ...

Débogage

EDIs, Navigateurs (intégrés ou extensions)

Intro. à HTML & CSS

HTML

HTML (Hypertext Markup Language) est un langage de balisage créé pour donner de la structure et un sens au contenu d'une page web.

CSS

CSS (Cascading Style Sheets) est un langage de présentation créé pour définir la présentation visuelle du contenu d'une page web.

Intro. à HTML & CSS

Termes HTML

Eléments: désignent des objets dans une page. Exemples: h1, p, a, div, em.
Balises: définissent la structure des éléments. Généralement un élément est constitué de deux balises: une balise ouvrante et une balise fermante. Exemple: 
contenu de l'élément h1
Attributs: propriétés utilisées les balises pour donner des instructions supplémentaires aux éléments. Exemple: 
contenu de l'élément h1

Intro. à HTML & CSS

Structure d'une page HTML

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Hello World</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>This is a website.</p>
  </body>
</html>
  • doctype: version HTML utilisée ('html' pour HTML5)
  • html: le document
  • head: titre, métadonnées (non visibles sur la page)
  • body: contenu (visible sur la page)

Intro. à HTML & CSS

Termes CSS

Sélecteurs: déterminent exactement à quels éléments les styles correspondants seront appliqués.
Propriétés: déterminent les styles qui vont être appliqués.
Valeurs: déterminent le comportement des propriétés correspondantes.
Exemple:
p {
  color: #ff0000;
  font-size: 16px;
}
Sélectionne tous les paragraphes ('p') et leur applique une couleur de texte rouge et une taille de police de 16 pixels.

Intro. à HTML & CSS

Ecriture longue vs. courte

Certains styles CSS peuvent être définis de différentes façons: en définissant les différentes propriétés séparément, une par ligne (écriture longue), ou en définissant toutes les propriétés en une seule ligne (écriture courte). Exemple: 
/* Ecriture longue */
p {
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 10px;
  padding-left: 20px;
}

/* Ecriture courte */
p {
  padding: 10px 20px;
}

INTRO. À HTML & CSS

Commentaires HTML

En HTML, des commentaires peuvent être ajoutés au code en le mettant entre <!-- et -->. Ce texte ne sera pas affiché dans la page. Exemple: 
<!-- commentaire HTML -->

Commentaires CSS

En CSS, des commentaires peuvent être ajoutés au code en le mettant entre /* et */. Ce texte ne sera pas interprété. Exemple: 
/* commentaire CSS */

INTRO. À HTML & CSS

Validation de Code

Le W3C met à disposition des développeurs des validateurs HTML et CSS. Ceux-ci analysent le code à la recherche d'erreurs. La validation nous aide à détecter les erreurs, à nous assurer que notre code sera affiché correctement par les navigateurs et à apprendre les bonnes pratiques à suivre.
Les validateurs sont accessible sur les liens suivants:
  • HTML : http://validator.w3.org
  • CSS : http://jigsaw.w3.org/css-validator

Sélecteurs CSS

Sélecteurs de type

Ciblent les éléments grâce à leurs types.
Exemple:
HTML CSS 
<p> ... </p>
 
/* Eléments de type paragraphe */
p {  }

Sélecteurs CSS

Sélecteurs d'ID

Ciblent les éléments grâce à leurs IDs. Un ID ne peut être affecté qu'une seule fois dans une page.
Exemple:
HTML CSS 
<div id="header">...</div>
 
/* L'élément avec
   id='header' */
#header {  }

/* L'élément 'div' avec
   id='header' */
div#header {  }

Sélecteurs CSS

Sélecteurs de classe

Ciblent les éléments grâce à leurs classes. Une classe peut être affectée plusierus fois dans une page.
Exemple:
HTML CSS 
<div class="comment">...</div>
<div class="comment even">...</div>
 
/* Tous les éléments avec
   class='comment' */
.comment {  }

/* Eléments 'div' avec
   class='comment' */
div.comment {  }

Sélecteurs CSS

Pseudo-classes

Les pseudo-classes sont utilisées pour cibler les éléments selon leurs états.
Exemples: 
/* Les liens visités */
a:visited {  }

/* Tous les 'div' au moment où l'utilisateur passe la souris dessus */
div:hover {  }

/* Parmis les paragraphes descendants d'un autre élément,
   cibler le premier */
p:first-child {  }

/* Parmis les paragraphes descendants d'un autre élément,
   cibler le dernier */
p:last-child {  }

Sélecteurs CSS

Sélection par attributs

Il est possible de cibler les éléments selon les valeurs de leurs attributs.
Exemples: 
/* Tous les paragraphes qui possèdent un attribut 'title' */
p[title] {  }

/* Tous les champs de formulaire dont l'attribut 'type'
   est égal à 'text' */
input[type=text] {  }

Sélecteurs CSS

Combinaisons de sélecteurs

Il est possible de combiner plusieurs sélecteurs pour cibler les éléments avec plus de précision.
Exemples: 
/* Tous les paragraphes qui sont des descendants du 'div'
   dont l'ID est 'content' */
div#content p {  }

/* Tous les paragraphes qui sont des descendants DIRECTES
   du 'div' dont l'ID est 'content' */
div#content > p {  }

/* Tous les paragraphes et tous les 'div' de classe 'comment' */
p, div.comment {  }

Référencement du CSS

Dans une page HTML, il est possible de référencer du code CSS de trois façons différentes:
  • Fichier externe : le code CSS se trouve dans un fichier externe. Il est référencé par un élément link dans la section head du document. C'est la méthode préférée car elle sépare le HTML et le CSS et permet de réutiliser le fichier sur plusieurs pages.
  • Interne : le code CSS se trouve dans la page HTML, dans un élément style.
  • En ligne : le code CSS est définit dans l'attribut style de la balise à laquelle il va être appliqué.

Référencement du CSS

Exemples: 
<!-- Fichier externe (dans la section 'head' du document) -->
<link rel="stylesheet" href="file.css">

<!-- Interne -->
<style type="text/css">
p {
  color: #f60;
  font-size: 16px;
}
</style>

<!-- En ligne -->
<p style="color: #f60; font-size: 16px;">...</p>

Eléments bloc vs. en ligne

Tous les éléments HTML sont soit des éléments bloc soit des éléments en ligne.

Eléments bloc

Ils commencent sur une nouvelle ligne et occupent tout l'espace horizontal disponible. Ils peuvent contenir aussi bien des éléments bloc que des éléments en ligne.

Eléments en ligne

Ils ne commencent pas sur une nouvelle ligne et entrent dans le flux normal du texte. Ils n'occupent que l'espace horizontal necessaire à leur rendu et ne peuvent contenir que des éléments en ligne.

Divisions & Spans

Les divisions <div> et les spans <span> agissent comme des contenaires génériques.
Ils ne possèdent pas de valeur sémantique.
Ils sont très importants car ils permettent d'appliquer du CSS d'une façon ciblée.

Divisions & Spans

Division

Elément bloc.
Sert généralement à identifier différentes parties d'une page pour aider dans la définition de son aspect visuel.

Span

Elément en ligne.
Sert généralement à identifier une petite section de texte à l'intérieur d'un élément bloc.

Divisions & Spans

Exemples

<!-- div -->
<div class="social">
  <p>Lorem ipsum dolor sit amet...</p>
  <p>Lorem ipsum dolor sit amet...</p>
</div>

<!-- span -->
<p>Lorem ipsum <span class="tooltip">dolor sit</span> amet...</p>

Eléments de typographie

Entêtes

Eléments bloc.
Existent en 6 niveaux: <h1>, <h2>, <h3>, <h4>, <h5>, <h6>.
Servent à diviser le contenu en sections et définir leur hirarchie.
Donnent un titre aux sections qui les suivent.
Utilisés par les moteurs de recherche pour déterminer l'importance d'un contenu sur la page.
IMPORTANT: ne pas utiliser pour définir la taille du texte.
Exemple: 
<h1>Entête niveau 1</h1>
<h2>Entête niveau 2</h2>
<h3>Entête niveau 3</h3>

Eléments de typographie

Paragraphes

Eléments bloc.
Servent à ajouter des paragraphes à un document.
Exemple: 
<p>Lorem ipsum dolor sit amet, consectetur adipisicing elit.
   Ut enim ad minim veniam, quis nostrud exercitation ullamco.</p>
<p>Duis aute irure dolor in reprehenderit in voluptate velit esse cillum
   dolore eu fugiat nulla pariatur. Excepteur sint occaecat cupidatat
   non proident.</p>

Eléments de typographie

Strong

Elément en ligne.
Utilisé pour donner de l'importance à un texte.
Affiché généralement en gras par les navigateurs.
IMPORTANT: Ne pas utiliser pour le formatage.
Exemple: 
<p>Avant de faire X il est <strong>nécessaire</strong> de faire Y avant.</p>

Eléments de typographie

Em

Elément en ligne.
Sert à marquer un texte sur lequel on veut insister.
Affiché généralement en italique par les navigateurs.
IMPORTANT: Ne pas utiliser pour le formatage.
Exemple: 
<p>Dans <em>HTML5<em>, ce qui était appelé contenu de <em>type bloc</em>
   est maintenant appelé contenu de <em>flux</em>.</p>

Hyperliens

Eléments en ligne.
Eléments clés du web (Hypertext).
Permettent d'atteindre des cibles en cliquant dessus.
Exemple: 
<a href="http://www.google.com">Google</a>

Hyperliens

Chemins absolus

Contiennent le chemin complet d'une page, incluant protocole et domaine.
Utilisés pour référencer des pages externes au site.
Exemple: 
<a href="http://www.google.com">Rechercher sur Google</a>

Hyperliens

Chemins relatifs

Ne contiennent ni protocole ni domaine, seulement le chemin de la page.
Sont résolus relativement à la page actuelle et son domaine.
Utilisés pour référencer des pages du même site.
Exemples: 
<a href="/about.html">A propos</a>
<a href="/pages/about.html">A propos</a>

Hyperliens

Adresses mail

Il est possible de définir des liens vers des adresses mail.
Un clique dessus ouvre le client mail par défaut de l'utilisateur.
L'adresse mail est ajoutée automatiquement comme destinataire.
Exemples: 
<a href="mailto:nom@exemple.com">Me contacter par mail</a>

Hyperliens

Nouvelle fenêtre

Par défaut, un hyperlien ouvre la page cible dans la même fenêtre et remplace la page originale.
Il est possible d'instruire le navigateur d'ouvrire la cible dans une nouvelle fenêtre en mettant l'attribut target sur _blank.
Exemple: 
<a href="http://www.google.com" target="_blank">Rechercher sur Google</a>

Hyperliens

Liens internes

Il est possible de définir des liens vers d'autres parties de la même page.
Exemples d'utilisation: créer des tables de contenu, lier différentes section d'un document entre elles.
L'élément qu'on veut mettre comme cible doit avoir un ID.
On utilise l'ID de la cible précédé de '#' comme référence.
Exemple: 
<p>Voir <a href="#exemple">l'exemple</a> pour plus de détails.</p>
<h2 id="exemple">Exemple d'utilisation des hyperliens</h2>

Eléménets HTML5 de structure

HTML5 introduit de nouveaux éléments dont la fonction principale est d'améliorer la sémantique.
Jusqu'ici, on utilisait principalement div pour structurer un document.
Les nouveaux éléments structurels nous permettent d'ajouter de la sémantique à la structure.

Eléménets HTML5 de structure

Header

Elément bloc.
Représente un groupe d'introduction pour une page ou section d'une page.
Exemples de contenu: entêtes, texte d'introduction, éléments de navigation, logo, formulaire de recherche, etc.
Exemple: 
<header>
  <h1>Titre du document</h1>
  <nav>
    <ul>
      <li><a href="index.html">Accueil</a></li>
      <li><a href="contact.html">Contacte</a></li>
    </ul>
  </nav>
</header>

Eléménets HTML5 de structure

Footer

Elément bloc.
Représente le pied de page ou de section d'une page.
Le contenu devrait être relatif à la section qui le contient.
Exemples de contenu: auteur, copyright, liens vers des documents en relation, etc.
Exemple: 
<footer>Copyright 2013 XYZ</footer>

Eléménets HTML5 de structure

Nav

Elément bloc.
Représente une section ayant des liens de navigation.
Reservée aux sections de navigation principales, comme une table de contenu de la page, navigation au sein du même site, etc.
Eviter d'y insérer des liens externe.
Exemple: 
<header>
  <h3>Table de contenu</h3>
  <nav>
    <ul>
      <li><a href="#introduction">Introduction</a></li>
      <li><a href="#html">HTML</a></li>
      <li><a href="#css">CSS</a></li>
    </ul>
  </nav>
</header>

Eléménets HTML5 de structure

Article

Elément bloc.
Représente du contenu autonome qui peut être distribué ou réutilisé indépendemment, par exemple dans un flux RSS.
Exemples de contenu: message de forum, article de journal ou magazine, article de blog, commentaire d'utilisateur, etc.
Exemple: 
<article>
  <header><h1>Titre de l'article</h1></header>
  <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Mauris
     volutpat semper nisi, eget pretium nibh imperdiet eget. Nullam eu
     convallis turpis. Sed convallis libero eu felis tempus mollis</p>
</article>

Eléménets HTML5 de structure

Section

Elément bloc.
Représente un groupement thématique de contenu.
Ne pas utiliser si le seul but est l'application de styles CSS, ceci est le rôles des divisions div.
Exemple: 
<article>
  <section>
    <header><h1>Introduction</h1></header>
    <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit...</p>
  </section>
  <section>
    <header><h1>HTML</h1></header>
    <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit...</p>
  </section>
</article>

Eléménets HTML5 de structure

Aside

Elément bloc.
Représente du contenu indirectement lié à ce qui l'entoure et peut en être séparé.
Exemples de contenu: glossaire, image, publicité, barre d'outils, liens externes en relation, etc.
Exemple: 
<article>
  <p>Cet été, nous avons visité le Louvre.</p>
  <aside>
    <h4>Louvre</h4>
    <p>Le plus grand musée de Paris.</p>
  </aside>
</article>

Valeurs CSS

Longueurs

Il existe plusierus unités en CSS pour définir les longueurs.
Certaines définissent des valeurs absolues, comme px pour pixels.
D'autres définissent des valeurs relatives, comme % pour pourcentage de la taille du parent et em pour fraction de la taille actuelle de la police.
Exemples: 
div {
  width: 50px; /* 50 pixels */
  width: 50%; /* moitié de la largeur du parent */
  padding-top: 0.5em; /* moitié de la taille actuelle de la police */
}

Valeurs CSS

Couleurs

Les couleurs en CSS peuvent être définies de différents façons.
Les plus répandues utilisent les valeurs des composants rouge, vert, bleu et alpha.
Chaque composant possède une valeur entre 0 et 255 (00 et FF en hexdécimal).
Exemples: 
div {
  color: #000000; /* noir */
  color: #FF0000; /* rouge */
  color: #00FF00; /* vert */
  color: #0000FF; /* bleu */
  color: rgb(0, 0, 255); /* bleu */
  color: rgba(0, 0, 255, 0.5); /* bleu à moitié transparent */
}

Valeurs CSS

Images

Certaines propriétés CSS acceptent des références d'images dans leurs valeurs.
Une référence d'image est définie par un chemin relatif ou absolu.
Un chemin relatif est résolu relativement à l'emplacement du fichier dans lequel la propriété est définie.
Exemples: 
div {
  /* chemin relatif */
  background-image: url('images/logo.png');

  /* chemin absolu */
  background-image: url('http://monsite.com/images/logo.png');
}

Le Modèle des Boîtes

Tout élément HTML est une boîte rectangulaire.
Le modèle des boîtes de CSS décrit ces boîtes et nous permet d'ajuster les marges, les bordures, les espacements et la taille de chaque élément.

Le Modèle des Boîtes

Height

Définit la hauteur d'un élément bloc.
Par défaut, dépend du contenu de l'élément.
Peut être fixée à une valeur précise.
Exemple: 
div {
  height: 100px;
}

Le Modèle des Boîtes

Width

Définit la largeur d'un élément bloc.
Par défaut, occupe tout l'espace horizontal disponible.
Peut être fixée à une valeur précise.
Exemple: 
div {
  width: 400px;
}

Le Modèle des Boîtes

Margin

Sert à définir l'espacement autour d'un élément.
Se trouve en dehors de la bordure et est transparent.
Valeur par défaut dépend de l'élément et du navigateur.
Exemples: 
div {
  /* t:20px r:20px b:20px l:20px */
  margin: 20px;

  /* t:20px r:10px b:20px l:10px */
  margin: 20px 10px;

  /* t:20px r:10px b:5px l:10px */
  margin: 20px 10px 5px;

  /* t:20px r:10px b:5px l:1px */
  margin: 20px 10px 5px 1px;
}
 
div {
  /* t:20px r:10px b:5px l:1px */
  margin-top: 20px;
  margin-right: 10px;
  margin-bottom: 5px;
  margin-left: 1px;
}

Le Modèle des Boîtes

Padding

Sert à définir l'espacement entre le contenu et la bordure.
Se trouve à l'intérieur de la bordure et prend la couleur de fond de l'élément.
Valeur par défaut dépend de l'élément et du navigateur.
Exemples: 
div {
  /* t:20px r:20px b:20px l:20px */
  padding: 20px;

  /* t:20px r:10px b:20px l:10px */
  padding: 20px 10px;

  /* t:20px r:10px b:5px l:10px */
  padding: 20px 10px 5px;

  /* t:20px r:10px b:5px l:1px */
  padding: 20px 10px 5px 1px;
}
 
div {
  /* t:20px r:10px b:5px l:1px */
  padding-top: 20px;
  padding-right: 10px;
  padding-bottom: 5px;
  padding-left: 1px;
}

Le Modèle des Boîtes

Border

Sert à définir une bordure autour d'un élément.
Se trouve entre padding et margin.
Une bordure possède trois propriétés: width, style et color.
Valeur par défaut: aucune bordure.
Exemples: 
div {
  /* Aucune bordure */
  border: none;

  /* Ligne continue */
  border: 3px solid #cccccc;

  /* Ligne discontinue */
  border: 3px dashed #cccccc;

  /* Ligne pointillée */
  border: 3px dotted #cccccc;
}
 
div {
  border-top: 3px solid #cccccc;
  border-right: 3px solid #cccccc;
  border-bottom: 3px solid #cccccc;
  border-left: 3px solid #cccccc;
}

Positionnement

CSS met à disposition les propriétés suivantes pour contrôler le positionnement d'un élément:

  • position : type de positionnement à utiliser.
  • top : décalage du bord gauche de l'élément.
  • right : décalage du bord droit de l'élément.
  • bottom : décalage du bord supérieur de l'élément.
  • left : décalage du bord inférieur de l'élément.

N.B. Les propriétés de décalage s'appliquent par rapport à différentes sources selon le type de positionnement.

Positionnement

Positionnement statique

C'est le type de positionnement utilisé par défaut.
L'élément est positionné selon son flux normal.
Sa position ne peut pas être modifiée et les propriétés de décalage n'ont aucun effet.
Exemple: 
div {
  position: static;
}

Positionnement

Positionnement relatif

L'élément est positionné relativement à sa position statique.
Les propriétés de décalage s'appliquent donc par rapport à la position statique.
Exemple: 
div {
  position: relative;
  top: 50px;
  left: 100px;
}

Positionnement

Positionnement absolu

L'élément est détaché du flux normal du document et n'influence plus le positionnement des autres éléments.
Les propriétés de décalage s'appliquent par rapport au premier parent ayant une position non statique, ou par rapport au document si un tel parent n'existe pas.
Exemple: 
div {
  position: absolute;
  width: 200px;
  height: 200px;
  top: 50px;
  right: 100px;
}

Positionnement

Positionnement fixe

L'élément est détaché du flux normal du document et n'influence plus le positionnement des autres éléments.
Les propriétés de décalage s'appliquent toujours par rapport au document.
L'élément reste fixe même lorsqu'on fait défiler le document.
Exemple: 
div {
  position: fixed;
  width: 200px;
  height: 200px;
  top: 50px;
  left: 100px;
}

Flottement

Il est possible de faire flotter un élément vers le côté droite ou le côté gauche de sont parent.
La position verticale d'un élément flottant est déduite de son flux normal, mais il est poussé horizontalement vers l'un des bords de son parent.
Des éléments flottants sur le même côté sont alignés horizontalement.
Les éléments non flottants sont alignés autour des éléments flottants.
Le flottement est définit avec la propriété float, qui peut avoir les valeurs left (à gauche), right (à droite) ou none (pas de flottement).

Flottement

Exemple

<style>
img {
  float: left;
}
</style>
<p>
  Some sample text in the first
  paragraph. It has a floating
  image right about here
  <img src="..."> ...
</p>
<p>
  The second paragraph is
  therefore also affected...
</p>

Formulaires

form

Un formulaire est définit avec l'élément bloc form.
Les champs du formulaire sont définis à l'intérieur de cet élément.
Attributs (les plus utilisés, tous optionnels) :
  • action : chemin du programme qui traitera les données.
  • method : méthode HTTP qui sera utilisée pour envoyer les données au serveur.
  • enctype : type MIME à utiliser pour encoder les données au serveur. La valeur par défaut est application/x-www-form-urlencoded. Pour l'envoi de fichiers, il faut utiliser multipart/form-data.

Formulaires

Méthode HTTP 'GET'

Données envoyées dans l'URL de la requête HTTP:
/test/form.php?name1=value1&name2=value2
Vu que les données font partie de l'URL, elle peuvent rester dans le cache et l'historique du navigateur et peuvent être ajoutées aux signets.
La taille des données est limitée.
Ne jamais utiliser pour les données sensibles.
L'utilisation correcte est pour effectuer des opérations qui ne modifient pas de données sur le serveur.

Formulaires

Méthode HTTP 'POST'

Données envoyées dans le corps de la requête HTTP.
Les données ne peuvent pas rester dans le cache et l'historique du navigateur et ne peuvent pas être ajoutées aux signets.
La taille des données n'est pas limitée.
L'utilisation correcte est pour effectuer des opérations qui modifient des données sur le serveur.

Formulaires

Champs

Les champs d'un formulaire sont définis avec les éléments en ligne input, textarea et select.
Attributs communs :
  • name : sert à identifier le champ dans les données envoyées au serveur. S'il n'est pas définit, les données du champ ne sont pas envoyées au serveur.
  • disabled : si présent, le champ est désactivé et l'utilisateur ne peux pas le manipuler. En plus, les données du champ ne sont pas envoyées au serveur.

Formulaires

Champ de texte simple

Définit avec l'élément input.
L'attribut type doit avoir la valeur text.
Attributs supplémentaires :
  • value : valeur du champ (texte prédéfinit).
  • readonly : si présent, le champ est ne peut pas être modifié par l'utilisateur.
  • maxlength : nombre maximum de caractères autorisé.

Formulaires

C'est un champ de texte simple dont le contenu est caché en affichant un caractère spécial à la place des caractères du texte.
Ce comprtement est activé en donnant la valeur password à l'attribut type de l'élément input.

Structure

Un tableau est définit avec l'élément table.
Un tableau est constitué de lignes. Une ligne est définie avec l'élément tr.
Chaque ligne est constituée de cellules d'entête ou de données. Une céllule d'entête est définie avec l'élément th. Une céllule de données est définie avec l'élément td.

Combinaison de céllules

Il est possible de combiner plusiseurs céllules (d'entête ou de données) en une seule.
Pour combiner horizontalement, on utilise l'attribut colspan.
Pour combiner verticalement, on utilise l'attribut rowspan.
On affecte à ces attributs le nombre de cellules à combiner.

Tableaux

Combinaison de céllules

Exemple: (le tableau à été stylé pour le rendre plus lisible!)
<table>
  <tr>
    <th>Société</th>
    <th>Contacte</th>
    <th>Pays</th>
  </tr>
  <tr>
    <th colspan="3">Partenaires Long Térme</th>
  </tr>
  <tr>
    <td rowspan="2">Alfreds Futterkiste</td>
    <td>Maria Anders</td>
    <td rowspan="2">Allemagne</td>
  </tr>
  <tr>
    <td>Philip Cramer</td>
  </tr>
  <tr>
    <td>Berglunds snabbköp</td>
    <td>Christina Berglund</td>
    <td>Suède</td>
  </tr>
  <tr>
    <th colspan="3">Nouveaux Partenaires</th>
  </tr>
  <tr>
    <td>Centro comercial Moctezuma</td>
    <td>Francisco Chang</td>

Styles CSS

En plus des propriétés CSS standards, il existe deux propriétés spécifiques à l'élément table:
  • border-collapse : définit le modèle de rendu des bordures. Valeurs possibles: separate (bordures séparées, valeur par défaut), collapse (bordures fusionnées) et inherit (hériter).
  • border-spacing : une longueur qui définit la distance entre les bords de cellules adjacentes pour le modèle de bordures séparées.

Styles CSS

Exemples:
table {
  border: 1px solid #c6c9cc;
  border-collapse: separate;
  border-spacing: 5px;
}

  padding: 8px;
  text-align: left;
  vertical-align: top;

  border-collapse: collapse;
